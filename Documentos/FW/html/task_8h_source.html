2184 /*
2185  * THIS FUNCTION MUST NOT BE USED FROM APPLICATION CODE. IT IS AN
2186  * INTERFACE WHICH IS FOR THE EXCLUSIVE USE OF THE SCHEDULER.
2187  *
2188  * THIS FUNCTION MUST BE CALLED WITH INTERRUPTS DISABLED.
2189  *

2191  * on the list of tasks waiting for a particular event, and the
2192  * list of delayed tasks. The task will be removed from both lists

2194  * there be no higher priority tasks waiting on the same event) or
2195  * the delay period expires.
2196  *
2197  * The 'unordered' version replaces the event list item value with the
2198  * xItemValue value, and inserts the list item at the end of the list.
2199  *

2201  * owning tasks priority) to insert the list item into the event list is task
2202  * priority order.
2203  *
2204  * @param pxEventList The list containing tasks that are blocked waiting
2205  * for the event to occur.
2206  *
2207  * @param xItemValue The item value to use for the event list item when the
2208  * event list is not ordered by task priority.
2209  *
2210  * @param xTicksToWait The maximum amount of time that the task should wait
2211  * for the event to occur. This is specified in kernel ticks,the constant
2212  * portTICK_PERIOD_MS can be used to convert kernel ticks into a real time
2213  * period.
